### 《理论地震学》第4次作业 - 波形反演震源机制

🚀 作业具体要求查看文件 **《第4次作业-波形反演》**；反演步骤流程参见详细的 **《简化CAP流程》**。

接下来说明作业文件夹中各个文件及目录的作用，帮助理清“波形反演确定震源参数”的工作流程。

#### 📁 目录与文件说明

##### 1. 核心数据与目标

- 20240226_Idaho/ (文件夹) —— 原始宽频带地震记录。

这是地震仪直接记录的原始数据。注意：原始数据通常不是位移！ 它们是机器记录的数字计数值（Counts），通常与地表振动的速度成正比，且包含了仪器自身的响应干扰。在本次作业中，老师已经帮我们预处理成了 disp，所以这个文件夹仅作科普和参考，不用参与实际的对比运算。

- disp/ (文件夹) —— 去除了仪器响应后的实际观测位移数据（Displacement）。

在地震学中，disp 文件夹通常代表已经完全处理好的“熟肉”数据。这意味着原始的机器计数值（Counts）已经被剔除了仪器自身的干扰，转换成了真实的物理位移（通常是米或厘米）。这是本次作业最核心的“标尺”数据，我们的最终目的就是调整正演参数，让计算出来的理论波形和这里的观测波形长得最像。

- evt_info.txt —— 地震定位信息与参考机制解。

这里面包含了地震发生的经纬度、以及官方或前人给出的震源参数参考值（走向 Strike、倾角 Dip、滑动角 Rake、深度 Depth）。我们在做“网格搜索”反演时，就是以这些参考值为中心，向外扩大范围去试探最佳解。

- Idaho_Crustal_Model.txt —— 一维地壳速度结构模型。

记录了地下不同深度的地震波传播速度（P波、S波速度）和岩石密度。正演程序必须知道这些地下的“交通路况”信息，才能模拟出地震波是怎么从地下震源传到地表的。

##### 2. 工具与程序包 

- fk/ (文件夹) —— FK (Frequency-Wavenumber) 理论地震图计算程序包的核心源码。

这是一个由专业地震学家编写的底层物理计算引擎，专门用来在给定的速度模型下计算“格林函数”。一般不需要修改里面的内容。

- forward_simulation_by_fk/ (文件夹) —— 正演模拟封装脚本。

这里面通常包含了调用底层 fk 引擎的代码。你输入参数（走向、倾角等），运行这里的脚本，它就会输出一条理论上的地震波形（理论图）。

- sac_read/ (文件夹) —— SAC 波形文件读取工具。
包含 readsac.m 等 MATLAB/Fortran 子程序，让我们的主程序能够读懂 .sac 格式的波形数据。
